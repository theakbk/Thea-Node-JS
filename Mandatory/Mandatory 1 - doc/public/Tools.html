Værktøjer

Node.JS

NodeJS er et open-source, crossplatform udviklingsmiljø, til at skrive backend serverside applikationer i Javascript.
Fordelen ved dette er, at med kun et sprog kan man bygge hele frontend og backenden. "Normal" Javascript på fronten og Node JS i back-enden.
Ligesom normal javascript kode, så kræver Node JS en fortolker som kan installeres her

Visual studio code

For at starte med at kode NodeJS, bør man først installere en IDE, i undervisningen har vi brugt Visual studio code.
Visual studio code er en crossplatform IDE, der understøtter de mest brugte sprog. Eksempler er Java, JavaScript, Node.js, Python og C++
Kan hentes her


Express

Express er et web application framework til Node JS. Det får ens applikation til at kunne komme på nettet. En Express-app er meget simpel at sætte op, og så er man igang. Se "den første webserver" for en gennemgang


Nodemon

Nodemon er en hjælperedskab der kører NodeJS koden automatisk, når end der kommer ændringer i koden. Hvis man ikke havde Nodemon, så skulle man manuelt kører koden hver gang man vil teste ændringerne. For at bruge nodemon skal det installeres og så bruger man en kommando fra terminalen, enten den indbyggede i IDE'en eller den indbyggede i computeren. Når man vil installere Nodemon skal man skrive "npm install -g nodemon" i terminalen, efter at have brugt kommandoerne cd og ls til at komme ind i projektet.


Postman

Postman er et hjælpeværktøj til udvikling. Postman er en applikation man kan installere eller bare bruge i browseren. Den hjælper udvikler til at teste koden imens den er under udvikling. Man behøver f.eks. ikke en HTML form, bare en postrequest i postman med et Json objekt.


NPM

NPM er den default packet manager som NodeJS bruger. En packet manager er god fordi den giver adgang til en række værktøjer, som gør at processen med at arbejde med de her ekstra pakker, blive automatiseret. NPM består af et command line klient, samt en større online database. Packet manageren kan man intstallere ved kommandoen "npm install" i kommando linjen. Dog skal man sikre sig man er inde i projektmappen.
